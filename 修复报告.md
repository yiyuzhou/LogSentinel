## 问题描述
用户反馈双击启动脚本后闪退，看到一堆错误信息。

## 遇到的错误

### 1. 编码问题（主要问题）
- **症状**: 批处理文件中的中文字符显示为乱码
- **原因**: 文件编码不匹配 - Windows CMD 需要 GBK 编码，但文件是 UTF-8 编码
- **错误信息**: 
  ```
  '锘緻 echo' 不是内部或外部命令
  '盷' 不是内部或外部命令
  文件名、目录名或卷标语法不正确
  ```

### 2. Python 依赖检查
- 启动脚本会自动检查并安装以下依赖：
  - flask
  - flask-cors
  - mysql-connector-python
  - paramiko
  - psutil

### 3. 配置文件检查
- 检查 `config\database.json` 是否存在
- 如不存在会提示用户复制配置模板

## 修复方案

### 方案 1: 简化启动脚本（推荐）
将启动脚本改为纯 ASCII 字符（英文），避免编码问题：

**修改内容**:
- 移除所有中文字符
- 使用英文提示信息
- 保留所有功能逻辑

**文件位置**:
- `D:\.openclaw\agents\weather-bot\workspace\export\video-task-system\start.bat`

### 方案 2: 使用正确的编码
如需保留中文，需要：
1. 使用 GBK 编码（代码页 936）保存批处理文件
2. 或使用 UTF-8 with BOM 编码

## 常见问题解决

### 端口 5000 被占用
```cmd
netstat -ano | findstr :5000
taskkill /F /PID <进程 ID>
```

### 依赖安装失败
```cmd
pip install flask flask-cors mysql-connector-python paramiko psutil
```

### 配置文件缺失
```cmd
cd config
copy database.json.example database.json
```
然后编辑 `database.json` 配置数据库连接信息。
