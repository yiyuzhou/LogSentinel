# 配置管理多配置保存功能修改说明

## 修改日期
2026-03-30

## 修改目标
实现配置管理功能中，选择某个配置后，"视频任务数据源"和"日志监控配置"在保存时都保存到当前选中的配置档（profile）下。

## 修改文件
- `D:\.openclaw\agents\weather-bot\workspace\scripts\settings.py`

## 主要修改内容

### 1. 数据结构调整

**修改前：**
```json
{
  "active_profile": "default",
  "profiles": {
    "default": {
      "name": "默认配置",
      "video": {...}
    }
  },
  "ssh_config": {...}  // 全局配置，不在 profile 内
}
```

**修改后：**
```json
{
  "active_profile": "default",
  "profiles": {
    "default": {
      "name": "默认配置",
      "video": {...},
      "ssh_config": {...}  // 每个 profile 独立的 SSH 配置
    },
    "production": {
      "name": "生产环境",
      "video": {...},
      "ssh_config": {...}
    }
  }
}
```

### 2. 后端 Python 代码修改

#### 2.1 DEFAULT_CONFIG 默认配置
- 添加了 `ssh_config` 字段到默认配置的 profile 中

#### 2.2 load_config() 函数
- 添加了旧配置自动迁移逻辑
- 如果检测到全局 `ssh_config`，自动迁移到每个 profile
- 迁移后保留全局配置以向后兼容

#### 2.3 get_ssh_config() 函数
- 修改为从当前激活的 profile 中获取 `ssh_config`
- 添加了向后兼容逻辑：如果 profile 中没有 `ssh_config`，尝试从全局获取

#### 2.4 save_ssh_config() 函数
- 添加了 `profile_name` 参数，支持指定保存到哪个 profile
- 默认保存到当前激活的 profile

#### 2.5 api_get_config() API
- 修改返回数据结构，每个 profile 现在包含 `ssh_config` 字段
- 移除了全局 `sshConfig` 返回字段
- 添加了向后兼容逻辑

#### 2.6 api_save_config() API
- 简化逻辑，直接保存前端传来的完整 profiles 数据
- 添加了日志输出，显示保存到哪个配置档

### 3. 前端 JavaScript 代码修改

#### 3.1 loadSettings() 函数
- 调用新增的 `loadProfileData()` 函数填充表单
- 同时加载 video 和 ssh_config 配置

#### 3.2 loadProfileData() 函数（新增）
- 从指定 profile 中加载 video 和 ssh_config 数据
- 填充到对应的表单字段
- 支持切换配置时快速更新表单

#### 3.3 switchProfile() 函数
- 修改为调用 `loadProfileData()` 直接更新表单，无需重新请求
- 更新 Toast 提示信息，显示同时切换了两个配置

#### 3.4 newProfile() 函数
- 新建配置时同时初始化 `video` 和 `ssh_config` 字段

#### 3.5 deleteProfile() 函数
- 优化删除确认提示，明确说明会同时删除两个配置
- 添加删除成功提示

#### 3.6 saveSettings() 函数
- **核心修改**：将 video 和 ssh_config 都保存到当前选中的 profile
- 更新成功提示信息，显示保存到哪个配置档

## 技术要点

### 向后兼容性
1. **旧配置迁移**：启动时自动检测全局 `ssh_config`，迁移到每个 profile
2. **配置读取**：如果 profile 中没有 `ssh_config`，回退到全局配置
3. **配置保存**：保留全局 `ssh_config` 字段（如果存在），但新代码不再使用

### 用户体验优化
1. **切换配置提示**：明确显示"视频任务数据源 + 日志监控配置"
2. **保存成功提示**：显示"已保存到配置档：XXX"
3. **删除确认**：明确列出将删除的配置项
